# macos_cleanup — "where_is_my_space" macOS cache/junk cleanup scripts

- `cleanup_all.sh` is the entry point: flushes DNS, rebuilds zsh completion cache, sources `cleanup_targets.sh`, then prunes Homebrew Caskroom installers, Docker, Trash, Atuin DB (VACUUM), and various app caches (GeForce NOW, Battle.net, Steam, DriveFS, GlobalProtect). Set `AGGRESSIVE=true` for the extra section (go/npm/nuget/docker prune + per-repo `git gc --aggressive`).
- `cleanup_targets.sh` is the declarative target list — mostly `try_clean`/`try_clean_pattern` calls grouped (Caches / Gaming / Development / Applications / System / Patterns). The Patterns section deletes `.DS_Store` (single `find ~` pass, prunes Library/.Trash/node_modules/.git, 30s `timeout`) and `.AppleDouble` files.
- `try_clean.sh` defines `try_clean <dir> <label> [exclude]` and `try_clean_pattern <base> <d|f> <pattern> <label>`; both size-measure, skip missing/empty dirs, and report. `helpers.sh` provides the spinner (`spinner_start`/`spinner_stop`) and `format_size`.
- `broken_links.sh` defines `report_broken_links` (called from the always-run "Dangling Symlinks" section): scans `$HOME` (pruning `Library`/`node_modules`/`.git`/`.Trash`/`.cache`/`.npm`) plus `/opt/homebrew/{bin,sbin,etc}` and `/usr/local/bin` for **dangling symlinks** (target deleted/renamed — e.g. a removed app's leftover link, or a renamed dotfiles target), using `find … -type l ! -exec test -e {} \;`. Report-only by default, flagging links into `~/projects/dotfiles` with `[dotfiles]`; set `PRUNE_BROKEN_LINKS=true` to `rm` them (a dangling symlink is pure junk). NB: "broken hardlinks" don't exist — a hardlink keeps the inode alive — so only symlinks are scanned.
- Invariant: cleanup is destructive but conservative — missing/empty/running-app targets are skipped silently, never failed. Safety options: `setopt no_unset pipe_fail` at the top (a typo'd variable can never expand to "" inside an rm path); `err_exit` is deliberately NOT set, matching the skip-silently design.
- Invoked three ways: locally via `up` (alias → `macos/update_all_and_cleanup_all.sh`, which runs `macos_update/update_all.sh` then `cleanup_all.sh` as **child zsh processes** — not sourced, so the safety setopts can't leak into the interactive shell); directly; or remotely via `clean.sh` (curl bootstrap that downloads the five sourced scripts to a tmpdir and sources `cleanup_all.sh` with `CLEANUP_DIR` set — safe there because the curl pipe runs in its own zsh process). It does its own `sudo -v`; `AGGRESSIVE=true up` still reaches it through the environment.
- Relation to `macos_update/`: `update_all.sh` does package/tool updates (brew, npm, uv, docker, etc.); this dir does only deletion. They are independent — `up` chains update → cleanup. Files are tracked by the repo `Makefile` (not symlinked).
