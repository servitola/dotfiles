# Travel Infographic

Генерирует PNG-сравнения и GIF-анимации маршрутов. Скрипты на Python с
PEP 723 inline-deps (uv сам ставит зависимости при первом запуске,
кэширует — повторные ~1 сек).

## Правило #1: рисовать только если полезно

**НЕ рисуй** когда:
- Один вариант и нет с чем сравнивать (для одного — только `route_map` или `route_gif`, не chart-ы).
- Запрос справочный ("во сколько отправляется поезд 097Э") — отдай текстом.
- У тебя нет реальных данных (не сочиняй цены чтоб нарисовать график).
- Разница между вариантами тривиальна и читается из текста ("оба варианта по 3 часа, разница 200₽").

**Рисуй** когда:
- Есть 2+ варианта с **неочевидной** разницей: дешёвый дольше door-to-door, дорогой быстрее но красноглазый, и т.п.
- Пользователь выбрал/обсуждает конкретный маршрут — покажи его на карте или анимацией.
- Гибкие даты ±несколько дней с разбросом цен.
- Многоплечевая поездка где итоговое время сильно отличается от суммы перелётов.

## Какой скрипт когда

Имя файла-результата подсказывает что это, чтобы пользователь сразу понимал:

| Файл | Скрипт | Когда |
|---|---|---|
| `route_<from>_<to>_animated.gif` | `gen_route_gif.py` | один конкретный маршрут, обсуждаем/предлагаем именно его |
| `route_<from>_<to>_map.png` | `gen_route_map.py` | один маршрут, но анимация избыточна (короткий перелёт без пересадок) |
| `compare_timeline.png` | `gen_timeline.py` | 2+ варианта, главный инсайт — door-to-door время с трансферами |
| `compare_routes.png` | `gen_pareto.py` | 2-5 вариантов маршрута — карточки в стиле Skyscanner с pre-leg деталями |
| `dates_calendar.png` | `gen_calendar.py` | юзер гибок ±5+ дней, цены гуляют |
| `cost_breakdown.png` | `gen_cost.py` | "дешёвый" билет на самом деле дорогой из-за багажа/места |

## Как вызывать

Каждый скрипт принимает `--data <json>` (либо путь к файлу, либо `-` для stdin) и `--out <path>`.

```bash
# Записать данные в JSON
cat > /tmp/trip.json << 'EOF'
{
  "title": "Лимасол → Армавир, 1 сентября",
  "options": [
    {
      "label": "Через Сочи + поезд",
      "legs": [
        {"type": "transfer", "label": "Дом → LCA", "hours": 1.5},
        {"type": "flight",   "label": "LCA → IST", "hours": 2.5},
        {"type": "layover",  "label": "Стамбул",   "hours": 4.0},
        {"type": "flight",   "label": "IST → AER", "hours": 3.5},
        {"type": "transfer", "label": "AER → жд", "hours": 1.0},
        {"type": "train",    "label": "Сочи → Армавир", "hours": 6.0}
      ],
      "total_hours": 18.5,
      "total_price_rub": 28000
    }
  ]
}
EOF

# Запустить
~/projects/dotfiles/claude-code/shared/skills/travel-infographic/gen_timeline.py \
  --data /tmp/trip.json \
  --out /tmp/compare_timeline.png
```

**Первый запуск** скрипта в новом окружении ставит deps через uv — это 30-60 сек. Дальше моментально.

## Схемы данных

### Timeline / Cost: список вариантов (старый формат)

```json
{
  "title": "string",
  "options": [
    {
      "label": "string (короткое имя варианта)",
      "legs": [{"type": "flight|train|transfer|layover|bus|car", "label": "string", "hours": float}],
      "total_hours": float,
      "total_price_rub": int,
      "cost_breakdown": {"base": int, "taxes": int, "bag": int, "seat": int}
    }
  ]
}
```

`legs` и `total_hours` обязательны для `gen_timeline`. `cost_breakdown` обязателен для `gen_cost`.

### Compare (gen_pareto.py): boarding-pass карточки

Богатый формат с временами вылетов, авиакомпаниями и ценами за плечо.

**Обязательно для каждого flight/train/bus сегмента:**
- `dep_time` и `arr_time` — **всегда местное + UTC в скобках**, формат `"HH:MM (HH:MM UTC)"`. Пример: `"07:30 (04:30 UTC)"`. Это обязательно для любых маршрутов, пересекающих часовые пояса — иначе невозможно проверить стыки и общую длительность. Для перелётов внутри одной таймзоны UTC можно опустить, но единообразие в одной инфографике важнее — лучше везде, чем местами.
- `from` и `to` — IATA код (LCA, EVN) ИЛИ русское название ("Ларнака", "Ереван"). Скрипт автоматически подставит второе по справочнику. Не комбинируй "EVN (Wizz)" — авиакомпанию клади в `carrier`, она появится между городами на линии.

**Желательно**: `carrier`, `duration_h`, `price_rub` — без них теряется ключевая инфа.

**Разделение времени туда / обратно.** Для маршрутов туда-обратно в опции
указывай `outbound_hours` и `return_hours` вместо суммарного `total_hours`.
Тогда вместо одной цифры «в пути» правый блок шапки покажет «туда X · обратно Y» — пользователю гораздо понятнее, особенно если ноги асимметричные (например, обратно с ночёвкой в хабе). Если есть ночёвка в хабе — добавь `overnight_note`, оно подпишется тонкой строкой ниже.

**Точка встречи (для совместных маршрутов).** Когда в одной инфографике
2+ человека, обязательно добавь сегмент `type: meet` в нужное место
итинерария — указать где, во сколько, и какой план («попутчик встречает пользователя в кафе у выхода А»). Это рендерится как янтарная плашка-«встреча», явный визуальный якорь — без неё пользователь спрашивает «а как мы стыкуемся?». Поля:

```json
{
  "type": "meet",
  "where": "Краснодар-1, кофейня Coffee Like у главного входа",
  "time": "~16:30 (13:30 UTC)",
  "note": "попутчик ждёт пользователя, дальше едут вместе Ласточкой 17:00"
}
```

**Note-плашки: `meet`, `warning`, `alt`, `tip`.** Все 4 типа имеют одинаковую
структуру (`where` — заголовок, `time` — необязательное время, `note` —
основной текст) и рендерятся одинаковым шаблоном-баннером с цветной рамкой
и заголовком вверху. Различаются только цветом и подписью:

| Тип | Цвет | Подпись | Когда использовать |
|---|---|---|---|
| `meet` | янтарный | ВСТРЕЧА | стык людей в одном маршруте |
| `warning` | розовый | ВНИМАНИЕ | риск опоздать / закрытый стык / визовые требования / тонкие моменты |
| `alt` | индиго | АЛЬТЕРНАТИВА | другой транспорт / другой отель / другой рейс на тот же сегмент |
| `tip` | изумрудный | СОВЕТ | полезная штука: чек-ин онлайн заранее, скачать посадочный, камера хранения и т.п. |

Используй их щедро в follow-along маршрутных гайдах — пользователь должен видеть в одной картинке и базовый план, и все боковые ветки/риски. Не используй в сравнительных инфографиках (там лишний шум) — только в детальной инфографике поездки.

**Для `label` варианта**: чистый текст без эмодзи (модель часто хочет добавить ✈️ 🇦🇲 — они не отрендерятся в Helvetica). Просто "через Ереван (EVN)".

**Лимит длины текстов** (иначе наезжают на ценовой блок или на названия городов):
- `label` варианта: **≤ 36 символов** (фит для FIG_W=10, fontsize=16, после ценового блока остаётся ~6.5 inches)
- `carrier` сегмента: **≤ 38 символов** (середина flight-строки, между сегментами IATA-blocks по 1.65 inch с краёв)
- если хочется больше деталей — выноси в `note` плашки `meet` или в layover `where`, там места куда больше

Длиннее — обрежется текстом или наедет на соседний блок. Проверяй сам: представь карточку 10" шириной, и каждый символ ≈ 0.18" в 16pt. Не уверен — сократи.

```json
{
  "title": "string",
  "options": [
    {
      "label": "через Ереван (EVN)",
      "total_hours": 8.5,
      "total_price_rub": 21000,
      "itinerary": [
        {
          "type": "flight",
          "from": "LCA", "to": "EVN",
          "dep_time": "07:30 (04:30 UTC)", "arr_time": "10:45 (06:45 UTC)",
          "carrier": "Wizz W6",
          "duration_h": 3.25,
          "price_rub": 8500
        },
        {"type": "layover", "where": "EVN", "duration_h": 2.5},
        {
          "type": "flight",
          "from": "EVN", "to": "KRR",
          "dep_time": "13:15 (09:15 UTC)", "arr_time": "16:00 (13:00 UTC)",
          "carrier": "Aeroflot SU",
          "duration_h": 2.75,
          "price_rub": 12500
        }
      ]
    }
  ]
}
```

Типы сегментов в `itinerary`: `flight`, `train`, `bus`, `car`, `layover`.
`layover` принимает `where` (город) и `duration_h`. Остальные —
полноценные сегменты движения.

### Calendar: даты + цены

```json
{
  "title": "string",
  "route": "LCA → AER",
  "year": 2026,
  "month": 9,
  "prices": {"2026-09-01": 28000, "2026-09-02": 31000, ...}
}
```

### Route map / Route GIF: один маршрут

```json
{
  "title": "Лимасол → Армавир (через Сочи)",
  "stops": [
    {"name": "Лимасол",  "type": "start"},
    {"name": "Ларнака (LCA)", "type": "airport"},
    {"name": "Сочи (AER)",   "type": "airport"},
    {"name": "Армавир", "type": "end"}
  ],
  "segments": [
    {"from_idx": 0, "to_idx": 1, "mode": "car",    "duration_h": 1.5, "label": "Такси"},
    {"from_idx": 1, "to_idx": 2, "mode": "flight", "duration_h": 3.0, "label": "Pegasus"},
    {"from_idx": 2, "to_idx": 3, "mode": "train",  "duration_h": 6.0, "label": "Поезд 643Г"}
  ],
  "total_hours": 10.5,
  "total_price_rub": 28000
}
```

Координаты `lat`/`lon` для известных городов берутся из `_cities.json`. Если города нет — добавь поле явно: `{"name": "Foo", "lat": 12.34, "lon": 56.78}`.

## Отправка в Telegram

После генерации обязательно:

- **PNG** → `mcp__bot__send_image` (inline превью в чате)
- **GIF** → `mcp__bot__send_animation` (auto-play, без скачивания)

Caption — короткая (1-2 строки) подпись с ключевым инсайтом, например:
"Через Сочи дольше на 4ч, но дешевле на 12000₽" или "Прямой только в купе, плацкарт через Челябинск".

## Палитра и стиль

Светлая, тёплая. Off-white фон #FAFAF7, белые карточки. Акценты:
- 🟢 Teal `#14B8A6` — перелёты
- 🟠 Orange `#F97316` — поезд/автобус
- 🟣 Violet `#8B5CF6` — трансферы/ожидание
- 💚 Emerald `#10B981` — лучший/самый дешёвый вариант
- 🟡 Amber `#F59E0B` — внимание (тайтовая пересадка, ночной рейс)
- 🌹 Rose `#F43F5E` — дорого / красноглазка / закрытый аэропорт

Цвета зашиты в `_theme.py`, не трогай их в data — скрипт сам подставит.

## Качество рендера

**Топ-качество всегда.** Все matplotlib-скрипты рендерят с `dpi=260` и
`bbox_inches="tight"` — текст должен быть резким даже после сжатия
Telegram. Если правишь существующие скрипты или добавляешь новый —
держи `dpi >= 240`, не ниже. Размытый текст = негодная инфографика.

Если пользователь жалуется на размытость — сначала проверь `dpi` в
конкретном скрипте (`grep dpi= gen_*.py`), потом думай про шрифты.

## Скрытие цен

Когда пользователь просит «без цен» / «follow-along» / маршрутный
гайд — в `gen_pareto.py` опусти поле `total_price_rub` (или поставь
`null`/`0`), и опусти `price_rub` в каждом сегменте `itinerary`.
Скрипт сам уберёт ценовой блок из шапки карточки и не нарисует цену
на отдельных плечах.

## Что НЕ делать

- Не использовать AI-генерацию картинок (gemini/fal/pollinations) для данных — цифры будут вымышленные.
- Не вызывать одновременно >1 скрипт без необходимости — пользователь видит **одну** картинку как главный визуал.
- Не выдумывать координаты — если города нет в `_cities.json`, добавь lat/lon явно.
- Не пихать GIF когда статичной карты хватает (анимация для маршрута >2 плечей, не для прямого перелёта).
