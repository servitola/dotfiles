# Claude Code Project Configuration

## Development Environment
- You are working on a macOS system
- Terminal is the primary interface
- Use modern development practices and ES6+ syntax where applicable

## Code Style Guidelines
- Use TypeScript when possible
- Follow ESLint and Prettier configurations
- Use meaningful variable and function names
- Add JSDoc comments for complex functions
- Prefer const/let over var
- Use arrow functions for short callbacks

## Project Structure
- Keep related files organized in logical directories
- Use index files for clean imports
- Separate concerns: components, utilities, types, etc.

## Git Workflow
- Use conventional commit messages
- Create feature branches for new work
- Keep commits focused and atomic
- Include co-authored-by when appropriate

## Testing Guidelines
- Write tests for new functionality
- Use Jest for JavaScript/TypeScript testing
- Aim for good test coverage on critical paths
- Test edge cases and error conditions

## Performance Considerations
- Minimize bundle size
- Use lazy loading where appropriate
- Optimize images and assets
- Consider memory usage in long-running processes

## Security Best Practices
- Never commit secrets or API keys
- Validate all inputs
- Use HTTPS for external requests
- Keep dependencies updated

## Documentation
- Update README.md for significant changes
- Document configuration options
- Include setup instructions for new developers
- Keep inline comments current

## Debugging
- Use console.log sparingly in production code
- Prefer debugger statements during development
- Use proper logging levels (info, warn, error)
- Include context in error messages

Remember: When in doubt, ask questions and explain your reasoning for implementation choices.
